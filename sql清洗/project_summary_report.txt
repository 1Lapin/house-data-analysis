一、项目概况
--------------------------------------------------------------------------------
项目名称: 长沙二手房数据清洗与可视化分析项目
数据时间: 2026年
数据来源: CSV文件（1500条原始记录）
覆盖区域: 岳麓、雨花、天心、开福、芙蓉、长沙县、望城、宁乡
项目目标: 建立企业级数据分层架构，完成数据清洗，支持Tableau可视化

二、数据架构
--------------------------------------------------------------------------------
【四层架构】
  ODS层（贴源层）: ods_house_raw
    → 原封不动存储原始数据，全VARCHAR字段
    → 数据量: 1500条
    → 用途: 审计追溯、数据恢复

  DWD层（明细层）: dwd_house_clean
    → 清洗后的单条房源记录，含质量标记
    → 数据量: 1492条有效 + 8条无效 = 1500条
    → 有效率: 99.5%
    → 用途: Tableau直接连接、明细查询

  DWS层（汇总层）: 
    → dws_district_stats（地区维度）
    → dws_roomtype_stats（房型维度）
    → dws_price_stats（价格段维度）
    → 用途: 预计算汇总，加速查询

  ADS层（应用层）:
    → ads_district_report（区域房价报表）
    → ads_budget_recommend（购房预算推荐）
    → ads_age_price_analysis（房龄分析）
    → ads_data_quality_daily（质量监控）
    → 用途: 业务方直接阅读

三、清洗规则
--------------------------------------------------------------------------------
【10条核心规则】
  R01 价格分级: 按总价分5档（低总价/中总价/高总价/豪宅/学区房）
  R02 房型解析: 正则提取室/厅/卫数
  R03 楼层编码: 低/中/高 → 1/2/3
  R04 朝向标准化: 提取主朝向，标记是否朝南
  R05 装修评分: 毛坯1/简装2/精装3/其他0
  R06 面积分段: <90/90-144/>144（对应契税标准）
  R07 房龄计算: 2026-建造年份，分4段
  R08 梯户解析: 中文数字转阿拉伯数字
  R09 密度计算: 户数/梯数，>4体验差，>15异常
  R10 异常标记: is_valid=0 + 记录原因（不删除）

【异常数据】
  梯户比例异常（梯数>户数）: 7条
  面积房型不匹配（3室<60㎡）: 1条
  合计无效: 8条（全部保留，仅标记）

四、核心发现
--------------------------------------------------------------------------------
【区域分析】
  最贵区域: 岳麓区（13,313元/㎡）
  最便宜区域: 宁乡（5,500元/㎡）
  性价比区域: 长沙县（7,500元/㎡，房源最多）

【价格分析】
  平均单价: 10,110元/㎡
  平均总价: 115.5万
  学区房: 11套，平均单价36,000元/㎡
  豪宅: 单价>3万或总价>400万

【房龄分析】
  次新房（6-10年）性价比最优
  新房（0-5年）溢价高但保值
  老房（>20年）需谨慎，关注拆迁

【朝向溢价】
  朝南房源占比: 75%
  朝南溢价: 9%（ vs 非朝南）
  南北通透均价最高

【装修影响】
  精装比毛坯贵约12%
  "其他"装修类型价格反而高（需进一步分析）

【楼层发现】
  低楼层反而略贵（可能含一楼花园/院子）
  中楼层、高楼层价格相近
