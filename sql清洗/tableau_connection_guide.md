# Tableau 连接配置指南

## 一、数据源连接

### 1.1 连接方式
- **数据源类型**: 文本文件（CSV）
- **文件路径**: `house_data_cleaned_for_tableau.csv`
- **编码**: UTF-8
- **分隔符**: 逗号

### 1.2 推荐连接字段映射

| Tableau 维度/度量 | 对应 CSV 字段 | 数据类型 | 说明 |
|:---|:---|:---|:---|
| 地区 | district | 字符串 | 地理角色设为"省/市/自治区" |
| 小区 | community | 字符串 | 可下钻到小区级别 |
| 价格分级 | price_level | 字符串 | 颜色编码用 |
| 房龄段 | age_range | 字符串 | 排序：新房→次新房→老房→老旧房 |
| 面积段 | area_range | 字符串 | 筛选器用 |
| 楼层 | floor_level | 字符串 | 排序：低楼层→中楼层→高楼层 |
| 朝向 | direction_clean | 字符串 | 金色标记含南朝向 |
| 装修 | decoration_type | 字符串 | 颜色编码用 |
| 总价 | total_price | 数值 | 默认聚合：平均值/中位数 |
| 单价 | unit_price | 数值 | 核心度量 |
| 面积 | total_area | 数值 | 筛选器用 |
| 房龄 | house_age | 数值 | 计算字段用 |
| 是否朝南 | is_south_facing | 数值(0/1) | 布尔筛选 |
| 装修评分 | decoration_score | 数值(1-3) | 排序用 |
| 居住密度 | density | 数值 | 异常值标记 |
| 是否有效 | is_valid | 数值(0/1) | **必须筛选 = 1** |

## 二、推荐工作簿设计

### 2.1 仪表板1：市场概览
**布局**: 顶部KPI + 中部地图 + 底部趋势

| 组件 | 类型 | 配置 |
|:---|:---|:---|
| 总房源数 | KPI卡 | COUNTD(house_id) |
| 平均单价 | KPI卡 | AVG(unit_price) |
| 平均总价 | KPI卡 | AVG(total_price) |
| 区域地图 | 填充地图 | 地区 → 颜色: AVG(unit_price) |
| 价格趋势 | 折线图 | 建造时间 → 单价 |

### 2.2 仪表板2：购房决策助手
**布局**: 左侧筛选器 + 右侧推荐列表

**筛选器面板**:
- 预算范围: 总价滑块 (0-1000万)
- 面积需求: 面积段多选框
- 房龄偏好: 房龄段多选框
- 朝向要求: 是否朝南单选
- 装修要求: 装修类型多选
- 地区偏好: 地区多选

**推荐列表**:
- 表格显示: 小区、总价、单价、面积、房龄、特色标签
- 按单价升序排列（性价比优先）

### 2.3 仪表板3：数据质量监控
**布局**: 全屏KPI + 异常明细表

| 组件 | 类型 | 配置 |
|:---|:---|:---|
| 有效率 | KPI卡 | SUM(is_valid)/COUNT(*) |
| 异常数量 | KPI卡 | COUNT(is_valid=0) |
| 异常分布 | 饼图 | invalid_reason |
| 异常明细 | 表格 | 原始字段 + 异常原因 |

## 三、关键计算字段（Tableau中创建）

```tableau
// 1. 单价分级（颜色编码）
IF [unit_price] < 7000 THEN "低价"
ELSEIF [unit_price] < 10000 THEN "中价"
ELSEIF [unit_price] < 15000 THEN "高价"
ELSE "豪宅/学区"
END

// 2. 性价比评分（综合指标）
([is_south_facing] * 0.3 + [decoration_score] * 0.2 + 
 (IF [house_age] <= 10 THEN 1 ELSE 0 END) * 0.3 +
 (IF [density] < 4 THEN 1 ELSE 0 END) * 0.2) * 100

// 3. 月供估算（假设首付3成，贷款30年，利率4.2%）
([total_price] * 0.7 * 10000) * 
(0.042/12 * POWER(1+0.042/12, 360)) / 
(POWER(1+0.042/12, 360) - 1)

// 4. 房龄状态
IF [house_age] <= 5 THEN "新房"
ELSEIF [house_age] <= 10 THEN "次新房"
ELSEIF [house_age] <= 20 THEN "老房"
ELSE "老旧房"
END
```

## 四、数据刷新策略

| 场景 | 操作 |
|:---|:---|
| 日常分析 | 直接连接CSV，手动刷新 |
| 定期报告 | 设置Tableau Server定时刷新 |
| 实时分析 | 建议迁移到MySQL，Tableau直连数据库 |

## 五、注意事项

1. **必须筛选 is_valid = 1**，否则异常数据会影响分析结果
2. **density 为 NULL 的记录**表示梯户比例无法解析，不影响分析
3. **price_level = '学区房'** 的样本量小（11套），分析时注意置信度
4. **全部房源都有电梯**，此字段无区分度，不建议作为筛选条件
